Context

We introduced a trace table (migration 0023) as a graph-native, hierarchical call tree for forensic tracing. Currently intent.trace_id points to record<trace>, enabling graph traversal from intents to their originating execution context.

However, the older subagent_traces field on message still stores execution traces as embedded arrays — denormalized, not independently queryable, and invisible to graph traversal.

Proposal

Migrate subagent_traces data from embedded array<object> on message into normalized trace records:

A subagent invocation becomes a root trace record (type: "subagent_spawn")
Each step (tool call, message, text) becomes a child trace with parent_trace pointing to the root
The message record links to the root trace via a relation edge (e.g. spawns relation) or a trace_id field
Trace hierarchy example

trace:root (type: "subagent_spawn", tool_name: "invoke_pm_agent")
  ├── trace:step1 (type: "tool_call", tool_name: "search_entities")
  ├── trace:step2 (type: "message")
  └── trace:step3 (type: "tool_call", tool_name: "create_observation")
Current embedded schema (to be replaced)

DEFINE FIELD subagent_traces ON message TYPE option<array<object>>;
DEFINE FIELD subagent_traces[*].agentId ON message TYPE string;
DEFINE FIELD subagent_traces[*].intent ON message TYPE string;
DEFINE FIELD subagent_traces[*].totalDurationMs ON message TYPE int;
DEFINE FIELD subagent_traces[*].steps ON message TYPE array<object>;
DEFINE FIELD subagent_traces[*].steps[*].type ON message TYPE string;
DEFINE FIELD subagent_traces[*].steps[*].toolName ON message TYPE option<string>;
DEFINE FIELD subagent_traces[*].steps[*].argsJson ON message TYPE option<string>;
DEFINE FIELD subagent_traces[*].steps[*].resultJson ON message TYPE option<string>;
DEFINE FIELD subagent_traces[*].steps[*].durationMs ON message TYPE option<int>;
DEFINE FIELD subagent_traces[*].steps[*].text ON message TYPE option<string>;
